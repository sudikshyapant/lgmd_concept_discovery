# Concept Discovery via Language-Guided Matrix Decomposition

Faithful re-implementation of *Interpretable Concept Discovery via Language-Guided
Matrix Decomposition* (ECCV 2026, #11398). Post-hoc concept discovery that replaces
the latent coefficient matrix of NMF with **CLIP-guided semantic activations**, so each
learned basis vector corresponds to a **named, human-interpretable concept**.

**Scope of this implementation:** a single ImageNet-1k class (`tabby cat`, n02123045)
with a **ResNet-50** backbone — the only intentional deviation from the paper (which uses
40 classes). Methodology, hyperparameters (`r = 25` concepts), baselines, and tests match
the paper.

## Concept vocabulary

Concepts are no longer generated by an LLM. They are read from `concept_vocab.json`
at the repo root — a table keyed by class name, where each class maps to an
over-provided list of candidate concepts (string labels, or `{"label": ...}` objects
with optional `id`/`category`/`description`). At run time `concepts.get_concepts`
runs the supplementary's two-stage filter and keeps exactly `r = 25`:

- **Stage 1 — lexical (suppl. A1.3):** keep 2–3 word concepts; drop generic filler
  terms (`object`, `scene`, `animal`, …) and concepts that overlap the class name —
  except concepts carrying a visual-attribute word (color/texture/parts/shape/
  pose/environment/material), which are kept.
- **Stage 2 — CLIP semantic (suppl. A1.4):** rank survivors by CLIP cosine
  similarity to the mean embedding of up to 100 class images (the prototype), then
  greedily keep diverse concepts, rejecting any with CLIP-text similarity `> 0.80`
  to one already kept.

Pass the class images to enable the relevance ranking (the notebook calls
`get_concepts(clip, images=train_imgs)`); without them the stored order is used.
Editing the file invalidates the concept cache automatically (its content hash feeds
`cache_name`). Add a new class by adding a key with ≥ `r` candidates and setting
`CONFIG["class_name"]` to match.

## Notes

- ICE/CRAFT/FACE all learn a basis `W` on training activations; validation reconstruction
  reuses LGMD's non-negative inference so methods are compared identically — matching the
  paper's "identical backbones, preprocessing, data splits, and concept counts r" (Sec 4).
  
  The only difference is how `W` is learned (ICE = NMF, CRAFT = recursive NMF, FACE = KL-reg NMF).
  
- **C-Ins** is reported as the normalized area under the concept-insertion curve (Sec 4.2):
  how fast the true-class prediction is restored as top-ranked concepts are added.
  
- Adding the **MobileNet** backbone or **Places365** only requires extending `CONFIG`.
